The only thing the type change is the initial configuration.

The types are:

Bare:
A virtual server with _no_ modules.

IPP Customer configuration:
A virtual server with these modules:
  • Contenttypes (map extension to MIME type)
  • Image-map support (<IMG ISMAP ...> tags)
  • The main RXML parser (parse RXML tags)
  • A Directory parser (generate directorylistings, and handle index.html etc.)
  • A filesystem (make files from your real filesystem available in Roxens virtual one)

Standard:
A virtual server with these modules:
  • Contenttypes (map extension to MIME type)
  • CGI script support (handles the CGI interface)
  • Image-map support (<IMG ISMAP ...> tags)
  • Pike script support (handle Pike scripts)
  • The main RXML parser (parse RXML tags)
  • A Directory parser (generate directorylistings, and handle index.html etc.)
  • A filesystem (make files from your hard-disk available)
  • A user database and security module (handle module-level security, and support for home-dirs for your normal UNIX users, via the user filesystem module)
  • A user filesystem (enables support for /~username/ URLs, the prefix (/~) can be configured.)

Proxy:
A server with all proxy and gateway modules enabled, but nothing else. Please note that no security is enabled per default in the proxy modules, which might not be desireably for your situation, since that allows anyone on the Internet to access your proxy.

Copy of..:
An exact copy of the mentined virtual server. You will have to change at least the listen ports.

This can be very useful, since you can make 'template' virtual servers (servers without any open ports), that you can copy later on.

For advanced users:
It is also possible to do this copying outside of the configuration interface, using a script to redesign the virtual server, and then restart roxen.
