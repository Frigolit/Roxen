User's Guide for AutoSite

AutoWeb

AutoWeb is the web site part of AutoSite.

AutoWeb Administration

Logging in

The AutoWeb administration interface is found by adding /webadm/ to the domain URL of your site in your browser, like this: http://www.domainname/webadm/

You will be asked for the administrator's username and password. If you give a correct username and password you will be asked to choose one of the following tabs: Template Editor, HTML Editor and Menu Editor. If one of the LogView modules is enabled a tab called Statistics or Simple Statistics will also be found.

Template Editor

This is where the layout of the site is configured.

Layout

Clicking on one of the headings will allow you to change the properties of the background, title, headings etc.

Navigation

Clicking on one of the headings will allow you to change the properties for the background, text style and colors of the navigation menus.

Template Functions

The Load... button will allow you to load a new template to the active template. The properties of the layout and navigation will be changed.

The New... button will create a new template that will be chosen as active template.

The Change... button will change the active template.

The Remove.. button will allow you to remove one of the templates if there is more than one.

HTML Editor

The HTML editor is where you find the file system of the site. Here you can create or upload new files, edit contents, moving or deleting existing files.

Handling Files

Click on the name of a file or directory to focus on the file or enter the directory.

Menu Editor

Here you can configure the menu file. Adding new items to the menu file is done by clicking the New... button and follow th instructions in the wizard or by chosing "Add to Menu" in the HTML Editor when focusing on a file or a directory.

If you want to move or delete a menu item first select it by clicking the corresponing radio button in the Select column, then click on one of the buttons Move Up, Move Down or Delete.

Statistics

This tab will be available when the LogView module is enabled for the site. Clicking on the tab you will be able to choose from various log analyzing tools.

Simple Statistics

This table will be shown when the Simple LogView module is enabled for the site. Choosing the tab will provide a graph showing the hitrate for the last month(s).

Creating HTML pages

The creation and handling of files is done in the HTML editor described above. There are a few aspects unique to AutoSite that should be considered when creating the HTML pages.

The <search> Tag

When the IntraSeek module is enabled for a site, the <search> tag can be used in the content files to add a search tool on the page. Simply put the <search> tag in the HTML/RXML code where you want the search tool to appear on your page.

The search tool will allow a vistor to search the content of the pages on the site.

<formsmailer>...</formsmailer>

Gtext Headings

The large headings, <h1></h1> and <h2&lgt;</h2&lgt; used in the content files on the site will be shown as gtext, making them widely configurable and nice looking. Since gtext is shown as gif images extensive use of gtext headings can make a page slow to load.

AutoMail

Roxen Mail is a complete mail system with a mail server, for this system the server is tifa.idonex.se, and a web based mail client, Roxen WebMail.

Different Protocols

There are a few different protcols used when fetching mail from a server using different methods of fetching and storing the mail. This system supports the two currently most popular protocols, POP and IMAP, and also provides its own web based mail client.

POP: Clients using POP copies the mail to the local file system. Most clients can be configured to leave a copy on the server, but when reading the mail with another client or from another computer the old mails will be shown as unread.

IMAP: The IMAP protocol stores the mail on the mail server which means that the information about mail that has been read etc is available using any IMAP client connecting to the server.

WebMail: Roxen WebMail is a web based mail client that can be used from your web browser without installing

POP3: Choosing server

If you read mail with a client using POP, choose tifa.idonex.se as POP server.

IMAP4: Choosing server

If you read mail with a client using IMAP, choose tifa.idonex.se as IMAP server.

Roxen WebMail

To use Roxen WebMail add /mail/ to the domain URL of your site like this: http://www.domainname/mail/

When asked you are asked for a username and a password type username*domainname as a username.

Listing Mailboxes

Clicking on this button you will se a table showing all your mailboxes. Click on the name to view the contents of a mailbox.

Reading Mail

To get new mail click on the List all mailboxes button and choose the Incoming mailbox. Unread mail is marked New! in the Flag column. Click on one of the text columns to read the mail.

Delete Mail

When focusing on a mail a

To delete a mail from a mailbox select it by clicking on the button in the Flag column and then click the Delete button. Several mails can be selected simultaneously so be alert that only the mails you really want to delete are seleted.

Write/Send Mail

To write a new mail click on the Compose Message button.

Create Mailbox

A folder called inbox is created automatically when a new user is added. When composing a new message a draft folder will be created and when the first message is sent a sent folder will be created.

If you want to create a new mailbox, click on the Create Mailbox button.

Delete Mailbox

Clicking on the Delete mailbox button you will be able to delete the mailbox of you choice. All mail currently in that mailbox will be deleted permanently.

User Settings

Clicking on this button will allow you to edit a signature file or change the personal data that is sent when sending mail.
